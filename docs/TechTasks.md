# Tech Stack Document — Hệ thống quản lí cho thuê phòng trọ

## Stack công nghệ

### Frontend

| Công nghệ | Miễn phí? | Cách hoạt động | Ứng dụng trong dự án |
|---|---|---|---|
| **React + React Router** | Có | React dùng Virtual DOM để re-render chỉ những component thay đổi thay vì reload toàn trang. React Router quản lí URL client-side — khi chuyển trang, chỉ component thay đổi, không gọi server để lấy HTML mới | Xây SPA với nhiều màn hình: Dashboard, Quản lí phòng, Hợp đồng, Hóa đơn, Thống kê. Mỗi màn hình là 1 route, điều hướng tức thì không blink |
| **Vite** | Có | Dùng ES Modules native của trình duyệt trong dev mode — browser tự import từng file, không bundle. Khi sửa 1 file, chỉ module đó được cập nhật (HMR). Build production dùng Rollup để bundle tối ưu | Dev server khởi động < 1 giây, sửa component hiện lên ngay lập tức. Build ra `dist/` để Nginx serve trong production |
| **Ant Design** | Có | Thư viện component React với ~60 component sẵn, có theme system dựa trên CSS-in-JS (v5). Mỗi component tuân theo design system nhất quán, hỗ trợ responsive và accessibility | Table phân trang cho danh sách phòng/hợp đồng/hóa đơn; Form + DatePicker cho nhập liệu; Modal.confirm cho xóa; Badge màu cho trạng thái phòng (`trong`/`cho_thue`/`dat_coc`/`sua_chua`); notification cho thông báo thành công/lỗi |
| **React Hook Form + Zod** | Có | React Hook Form dùng `ref` thay vì `state` để theo dõi giá trị form — không re-render component khi gõ phím. Zod định nghĩa schema kiểu TypeScript-first, khi `.parse()` sẽ throw lỗi chi tiết nếu dữ liệu không hợp lệ. Kết hợp qua `@hookform/resolvers/zod` | Form đặt cọc, làm hợp đồng, lập hóa đơn — validate ngay phía client trước khi gọi API. Schema Zod dùng chung với backend trong `shared/schemas/` — ví dụ rule "chỉ số điện mới ≥ chỉ số cũ" viết 1 lần, chạy được ở cả 2 phía |
| **TanStack Query (React Query)** | Có | Quản lí "server state" — dữ liệu đến từ API. Mỗi query có cache key, tự động refetch khi window focus lại hoặc sau khoảng thời gian `staleTime`. `useMutation` tự động invalidate cache sau khi POST/PUT/DELETE thành công | Dashboard: tự refetch KPI và cảnh báo định kỳ. Danh sách phòng/hợp đồng: không cần viết loading/error state thủ công. Sau khi thanh toán hóa đơn → tự cập nhật lại danh sách ngay lập tức |
| **Axios** | Có | HTTP client hỗ trợ interceptor — middleware chạy trước/sau mỗi request/response. Tự động serialize JSON, xử lý timeout, cancel request | Tạo 1 Axios instance với `baseURL` trỏ về Express server. Request interceptor đính kèm header chung. Response interceptor bắt lỗi 4xx/5xx → hiện toast thông báo lỗi tập trung thay vì xử lý ở từng component. Dùng cho mọi API call trong toàn bộ frontend |
| **Day.js** | Có | Thư viện xử lý ngày tháng nhẹ (~2KB gzip), API giống Moment.js. Immutable — mỗi thao tác trả về object mới. Hỗ trợ plugin: `duration`, `isBetween`, `isSameOrBefore` | Tính pro-rata tháng đầu: `(số ngày ở) / (tổng ngày trong tháng) × giá thuê`. Đếm tháng nợ liên tiếp. Kiểm tra hợp đồng sắp hết hạn trong 30 ngày cho dashboard cảnh báo. Format ngày hiển thị trên PDF/bảng |
| **Recharts** | Có | Thư viện biểu đồ xây trên D3.js nhưng bọc thành React component thuần. Mỗi loại biểu đồ là 1 component (`<LineChart>`, `<BarChart>`) với dữ liệu truyền qua props | Trang thống kê: BarChart doanh thu theo tháng/quý/năm. LineChart tỉ lệ lấp đầy phòng theo thời gian. PieChart phân bố trạng thái phòng theo khu |

### Backend

| Công nghệ | Miễn phí? | Cách hoạt động | Ứng dụng trong dự án |
|---|---|---|---|
| **Node.js + Express** | Có | Node.js chạy JavaScript ngoài trình duyệt, event loop non-blocking cho phép xử lý nhiều request đồng thời không cần thread. Express là framework tối giản: định nghĩa route + middleware chain — mỗi request đi qua chuỗi middleware theo thứ tự | REST API server xử lý ~35 endpoint cho toàn bộ nghiệp vụ. Middleware pipeline: Morgan → cors → helmet → JSON parser → Zod validate → Controller → Service → MongoDB |
| **Mongoose** | Có | ODM (Object Document Mapper) — ánh xạ MongoDB document sang JavaScript object có schema. Khi `.save()`, Mongoose validate theo schema trước khi ghi. Hỗ trợ middleware (pre/post hook), virtual field, populate (join thủ công) | Định nghĩa 14 collection với ràng buộc kiểu dữ liệu, enum, required. Pre-save hook trong `Phong` tự tạo bản ghi `LichSuGiaThuPhong` khi `gia_thue` thay đổi. Unique index trên `(ten, khu_id)` và `cmnd` |
| **Zod (shared schemas)** | Có | Zod chạy được cả trên Node.js lẫn browser. Schema định nghĩa 1 lần trong `shared/schemas/`, import vào cả 2 phía. Trên backend, tạo Express middleware: `req.body = schema.parse(req.body)` — nếu fail thì trả 400 kèm thông báo lỗi cụ thể | Validate request body cho tất cả POST/PUT endpoint: tạo hóa đơn (chỉ số điện/nước phải ≥ 0, ngày hợp lệ), làm hợp đồng (ngày kết thúc > ngày bắt đầu), thanh lý (tiền bồi thường ≥ 0). Cùng schema với frontend, lỗi validation hiển thị đồng nhất cả 2 phía |
| **cors** | Có | Middleware Express thêm header `Access-Control-Allow-Origin` vào response. Trình duyệt chặn request từ origin khác theo Same-Origin Policy — cors cho phép cấu hình whitelist origin nào được phép gọi API | Trong dev: React chạy port 5173, Express chạy port 3000 — khác origin, cần cors. Cấu hình `origin: process.env.CLIENT_URL` để chỉ cho phép frontend của dự án. Production: Nginx reverse proxy cùng domain nên cors ít quan trọng hơn |
| **dotenv** | Có | Đọc file `.env` vào `process.env` khi khởi động. Không bao giờ commit `.env` vào git — mỗi môi trường (dev, production) có giá trị khác nhau | Quản lí: `MONGODB_URI` (chuỗi kết nối DB), `PORT` (cổng server), `CLIENT_URL` (origin cho cors). Trong Docker Compose, các biến này inject vào container qua `environment:` field thay vì file `.env` |
| **Helmet** | Có | Middleware Express tự động thêm ~15 HTTP security header vào mọi response: `X-Content-Type-Options`, `X-Frame-Options`, `Content-Security-Policy`, `Strict-Transport-Security`... Ngăn các loại tấn công phổ biến ở tầng HTTP header | Dù hệ thống chỉ dùng trên LAN, việc thêm helmet là 1 dòng code và bảo vệ khỏi: clickjacking (X-Frame-Options), MIME sniffing (X-Content-Type-Options), XSS qua CSP. Đặt ngay đầu middleware chain |
| **Multer** | Có | Middleware Express xử lý `multipart/form-data` — định dạng của HTML form khi upload file. Parse binary data từ request stream, lưu tạm vào RAM (`memoryStorage`) hoặc disk (`diskStorage`) | Tính năng Backup/Restore (T9.2): màn hình Cài đặt cho phép upload file backup `.gz` → Multer nhận file → server chạy `mongorestore` từ file đó. Không có Multer thì không thể nhận file upload qua API |
| **Archiver** | Có | Thư viện tạo file nén (`.zip`, `.tar.gz`) theo dạng stream — không cần tạo file tạm trung gian, pipe thẳng vào `res` của Express | Tính năng Backup (T9.2): server chạy `mongodump` → output pipe qua Archiver nén thành `.tar.gz` → pipe thẳng vào HTTP response → trình duyệt tải xuống. Dung lượng file backup nhỏ hơn nhiều so với dump thô |
| **Winston + Morgan** | Có | Morgan là middleware Express, ghi log mỗi HTTP request (method, URL, status code, thời gian phản hồi) theo format chuẩn. Winston là logger đa level (`error`, `warn`, `info`, `debug`), hỗ trợ nhiều transport (console, file) đồng thời | Morgan: log tất cả request ra console khi dev, ghi file `logs/access.log` khi production. Winston: bắt lỗi từ Express error handler trung tâm, ghi `logs/error.log` (chỉ lỗi) và `logs/combined.log` (tất cả). Khi deploy LAN có sự cố, xem log để debug |
| **Puppeteer** | Có | Thư viện điều khiển Chromium (hoặc Chrome) ở chế độ headless qua DevTools Protocol. `page.goto()` load URL, `page.pdf()` chụp trang thành PDF với độ chính xác pixel hoàn toàn giống print từ trình duyệt | 4 loại PDF: hợp đồng thuê phòng, hóa đơn tháng, biên bản thanh lý, biên bản hủy hợp đồng. Mỗi loại có HTML template với dữ liệu inject vào. API `GET /api/in/:loai/:id` trả về file PDF để trình duyệt mở hoặc tải |
| **ExcelJS** | Có | Thư viện tạo/đọc file `.xlsx` theo chuẩn OOXML. Hỗ trợ định dạng cell (font, màu, border, merge), thiết lập độ rộng cột, nhiều sheet | 3 loại báo cáo Excel: doanh thu theo kỳ, danh sách nợ khách hàng, công suất phòng theo tháng. API `GET /api/xuat/:loai?format=excel` stream file `.xlsx` về trình duyệt |

### Infrastructure & DevOps

| Công nghệ | Miễn phí? | Cách hoạt động | Ứng dụng trong dự án |
|---|---|---|---|
| **MongoDB (Community)** | Có | Cơ sở dữ liệu NoSQL lưu dữ liệu dạng BSON document (giống JSON). Không có schema cứng ở tầng DB — schema do Mongoose quản lí. Hỗ trợ index, aggregation pipeline, transaction | Lưu 14 collection. Dùng aggregation pipeline cho dashboard KPI (tổng doanh thu, tỉ lệ lấp đầy) và báo cáo thống kê. Backup bằng `mongodump` → file BSON có thể restore chính xác |
| **Nginx** | Có | Web server / reverse proxy hiệu năng cao. Trong production, serve static file (HTML/JS/CSS) cực nhanh từ filesystem. Có thể làm reverse proxy: nhận request → forward đến Express server | Trong Docker Compose: container `client` dùng Nginx serve React build (`dist/`). Nginx nhận request, nếu path bắt đầu `/api` thì proxy_pass đến container `server` (Express), còn lại serve `index.html` (SPA routing) |
| **Docker Compose** | Có | Định nghĩa và chạy nhiều container Docker cùng lúc qua 1 file `docker-compose.yml`. Các container chia sẻ network nội bộ, có thể gọi nhau bằng tên service | 3 service: `mongo` (MongoDB với volume persistent giữ data), `server` (Node.js/Express), `client` (Nginx serve React). Chủ trọ chỉ cần cài Docker và chạy `docker compose up -d` — toàn bộ app chạy trên máy LAN |
| **PM2** | Có | Process manager cho Node.js. Chạy app dưới dạng daemon, giám sát process, tự restart khi crash. Cluster mode: tạo N worker process = số nhân CPU, load balance request | Chạy Express server trong production (bên trong Docker container hoặc trực tiếp). `ecosystem.config.js` cấu hình cluster mode, tự restart, ghi log process ra file. `pm2 monit` để xem realtime CPU/RAM |
| **nodemon** | Có | Watcher theo dõi thay đổi file `.js`/`.ts` trong thư mục server, tự restart Node.js process khi có file thay đổi | Dev dependency: chạy `nodemon server/index.js` thay vì `node server/index.js` — sửa file backend → server tự restart sau < 1 giây, không cần Ctrl+C và chạy lại thủ công |
| **concurrently** | Có | Chạy nhiều npm script song song trong 1 terminal, prefix output mỗi process bằng màu khác nhau để phân biệt | Dev workflow: 1 lệnh `npm run dev` ở root chạy cả `nodemon server/` lẫn `vite client/` cùng lúc, output phân biệt màu `[server]` và `[client]` — thay vì phải mở 2 terminal riêng |
| **ESLint** | Có | Static analysis tool phân tích AST (cây cú pháp) của code JavaScript/JSX, phát hiện lỗi tiềm ẩn và vi phạm coding convention mà không cần chạy code | Bắt lỗi thường gặp trong dự án: dùng biến chưa khai báo, quên `await` khi gọi async function, `useEffect` thiếu dependency, import không dùng. Chạy trong CI hoặc pre-commit để ngăn commit code lỗi |
